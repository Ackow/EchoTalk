# Global Tech Inc. Senior Frontend Developer Interview Preparation Guide

This guide contains core knowledge base details for evaluating candidates applying for the Senior Frontend Developer position.

## Technical Core Concepts

### 1. React Render Lifecycle and Virtual DOM
- Virtual DOM is a lightweight JavaScript representation of the real DOM.
- React uses a reconciliation process called Fiber to update the real DOM efficiently.
- Re-rendering is triggered by state changes (`useState`) or prop changes.
- To prevent unnecessary renders, developers should use `React.memo` for component memoization, `useCallback` to cache callback functions, and `useMemo` to memoize expensive computations.

### 2. Browser Rendering Path
- Critical Rendering Path (CRP) steps: DOM -> CSSOM -> Render Tree -> Layout -> Paint.
- Repaint happens when changes affect the visibility of elements but not their geometry (e.g., color, visibility).
- Reflow (Layout) happens when changes affect the layout of the page (e.g., width, height, position). Reflow is much more expensive than repaint.
- CSS transitions and transforms (e.g., `transform: translate3d`) can bypass Layout and Paint steps, running directly on the GPU Compositor layer for optimal performance.

## Behavioral STAR Method Alignment
Candidates are evaluated using the STAR technique:
- **Situation**: Establish the context of your project.
- **Task**: Describe the challenge or problem.
- **Action**: Explain the exact engineering steps you took.
- **Result**: Highlight the outcome with quantifiable metrics (e.g., improved page load speed by 30%).
